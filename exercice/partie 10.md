# Partie 10 : Readme
Notre projet est fini ! Bravo !

Mais... Ce n'est pas totalement fini pour autant !

Dans la vraie vie, une fois que votre projet en développement est terminé, il faut :
- Le partager avec votre équipe, pour faire des tests, continuer à développer,
- le mettre en ligne (en production),
- le publier dans un dépôt open-source si vous le souhaitez, pour partager votre travail avec d'autres collaborateurs,
- ...

Sauf que vos collègues, vos collaborateurs et vous-même dans un futur proche, ne savez pas du tout ce qu'il faut faire pour que l'installation puisse marcher...

- Comment installer et démarrer votre projet ?
- Est-il nécessaire de renseigner des variables d'environnement comme dans le config.php ?
- Est-il nécessaire d'avoir une base de données ? 
- Si oui, où peut-on trouver le script ?
- etc.
- Des comptes utilisateurs sont-ils déjà présents ? 
- Si oui, quels sont les identifiants pour se connecter ?

Et toute autre information qui pourrait être utile à l'exploitation de votre projet (version des langages, librairies utilisées, etc.)

Aussi, on ajoutera toujours un peu de **contexte** à propos du projet. Vous faites des projets relativement complexes, alors parlez-nous-en ! Les futurs recruteurs apprécieront de comprendre le contexte et l'ampleur du projet réalisé.
- Quel était le cahier des charges ? 
- Quelles étaient les contraintes de temps, d'espace, d'équipe, ... ?
- Quels ont été les partis-pris face à telle ou telle autre difficulté ? Pourquoi ces choix ?

Pour finir, le README est avant tout utile à vous ! D'ici quelques semaines, vous aurez oublié comment démarrer ce projet, le README est là pour en garder une trace.
